# Lista de entrega

## Codigo y ejecucion

- [ ] `docker compose config --quiet` correcto.
- [ ] API y frontend construyen desde cero.
- [ ] Version visible `0.16.0-rc1` o version final.
- [ ] No hay secretos reales en `.env.example`.
- [ ] Los cuatro perfiles inician sesion.

## Validacion

- [ ] Pruebas backend superadas.
- [ ] Build frontend superado.
- [ ] Matriz de permisos superada.
- [ ] Calidad de VetIA ejecutada en retrieval.
- [ ] Calidad con generacion ejecutada al menos una vez.
- [ ] Validacion del MVP revisada.
- [ ] Smoke test con monitorizacion superado.
- [ ] Backup creado y verificado.
- [ ] Restauracion probada en una copia o entorno limpio.

## Demostracion

- [ ] Dataset reconstruido con `reset-demo.ps1`.
- [ ] Avisos y casos recurrentes visibles.
- [ ] Fuentes externas descargadas o limitacion documentada.
- [ ] Evidencias generadas y sus hashes comprobados.
- [ ] Guion ensayado en menos de 15 minutos.

## Documentacion

- [ ] Arquitectura final.
- [ ] Modelo de datos.
- [ ] Endpoints y matriz de permisos.
- [ ] Instalacion, operacion, backup y troubleshooting.
- [ ] Memoria de maximo 20 paginas.
- [ ] Limitaciones y trabajo futuro.

## Congelacion

A partir de este candidato no se incorporan modulos de negocio. Solo se aceptan
correcciones reproducibles y documentadas antes de etiquetar `v1.0.0`.
