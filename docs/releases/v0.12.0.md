# GamuCare AI v0.12.0

## Objetivo

Cerrar el bloque de DevOps y operacion del MVP local: observabilidad, alertas
tecnicas, logs, backups, restauracion, smoke tests y pipeline reproducible.

## Cambios funcionales

- Nueva pantalla **Estado tecnico** para `clinic` y `staff`.
- Estado y latencia de PostgreSQL, Qdrant y Ollama.
- Accesos directos a Grafana, Prometheus y Alertmanager.

## Cambios backend

- Endpoint publico `/health/dependencies`.
- Endpoint autenticado `/api/v1/observability/status`.
- Metricas de VetIA, Ollama, tokens, Qdrant y dependencias.
- Logs JSON opcionales con `RotatingFileHandler`.
- Dos pruebas nuevas de dependencia y version actualizada.

## Infraestructura

- PostgreSQL Exporter y Blackbox Exporter.
- Prometheus con reglas y retencion de 15 dias.
- Alertmanager local.
- Dashboard Grafana ampliado.
- Rotacion del driver de logs Docker en todos los servicios.

## Backups

- `backup.ps1`, `verify-backup.ps1` y `restore.ps1`.
- `pg_dump` custom, snapshot de Qdrant, `data.zip` y manifiesto SHA-256.

## CI/CD

- Cobertura minima del 55%.
- Artefactos JUnit, cobertura y frontend compilado.
- Validacion del Compose base y del perfil monitoring.
- Cache de Buildx.
- Publicacion de imagenes versionadas en GHCR al crear una etiqueta `v*`.

## Base de datos

No hay cambios de esquema. La migracion de cabecera continua siendo
`0009_system_evaluation`.

## Actualizacion

```powershell
.\scripts\upgrade-v0.12.ps1
```

Parametros opcionales: `-SkipMonitoring`, `-SkipTests` y `-SkipEvaluation`.

## Reversion

La version no cambia el esquema. Para volver a la v0.11.0, restaura los ficheros
de esa entrega y reconstruye `api` y `web`. Los datos siguen siendo compatibles.
