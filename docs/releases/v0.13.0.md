# GamuCare AI 0.13.0

## Objetivo
Separar completamente la operacion clinica de las funciones de administracion tecnica.

## Cambios funcionales
- Nuevo perfil `technical`.
- Menu tecnico exclusivo con Calidad de VetIA, Validacion del MVP, Integracion Wakyma, Auditoria, Seguridad y Estado tecnico.
- Los perfiles clinic, staff y owner dejan de visualizar esas opciones.
- El perfil tecnico es redirigido a Estado tecnico al iniciar sesion.
- Nuevo usuario local `technical@gamucare.local`.

## Cambios de seguridad
- Las rutas tecnicas quedan restringidas en FastAPI al rol `technical`.
- Dashboard, mascotas, planes y VetIA quedan bloqueados para el rol tecnico tanto en frontend como en API.
- La integracion Wakyma solo puede validarse o ejecutarse desde el perfil tecnico.

## Base de datos
No hay cambios de esquema ni nueva migracion Alembic. El script de actualizacion crea o normaliza la cuenta tecnica mediante `app.scripts.ensure_technical_user`.

## Compatibilidad
El script `upgrade-v0.13.ps1` conserva los volumenes, es compatible con Windows PowerShell 5.1 y reutiliza la monitorizacion existente.

## Credenciales de demostracion
- Usuario: `technical@gamucare.local`
- Contrasena inicial: `GamuCare123!`

En entornos no demostrativos debe definirse `TECHNICAL_INITIAL_PASSWORD` antes de ejecutar la actualizacion.
