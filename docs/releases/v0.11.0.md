# GamuCare AI v0.11.0

## Objetivo

Convertir las comprobaciones dispersas del MVP en una evaluacion formal,
repetible y defendible durante la presentacion del TFM. La version tambien hace
visible el numero de release y cambia el nombre comercial del asistente a
**VetIA** en la interfaz.

## Cambios funcionales

- Nueva opcion **Validacion del MVP** para `clinic` y `staff`.
- Ejecucion de una suite completa reservada a `clinic`.
- Visualizacion de puntuacion global, criterios superados, cobertura, exactitud
  de avisos, controles de seguridad, recuperacion de VetIA y latencia P95.
- Descarga del informe de evaluacion en Markdown.
- Version `v0.11.0` debajo del logotipo en login y menu lateral.
- Sustitucion de las etiquetas visibles "RAG" por **VetIA**.

## Cambios de backend

- Modelo `SystemEvaluationRun`.
- Router `/api/v1/quality`.
- Servicio `SystemEvaluator`.
- Script `python -m app.scripts.evaluate_system`.
- Dataset `acceptance_criteria_v1.json` con 16 criterios.
- Dataset `alert_cases_v1.json` con ocho casos positivos y negativos.
- Informes JUnit y cobertura de pytest.
- Metricas Prometheus:
  - `gamucare_quality_evaluations_total`
  - `gamucare_quality_evaluation_duration_seconds`

## Base de datos

Migracion Alembic `0009_system_evaluation`:

- Crea `system_evaluation_runs`.
- Guarda version, estado, pruebas, cobertura, metricas, detalle e informe.
- No modifica datos de clientes, mascotas, planes, pagos ni historiales.

## Pruebas

- 54 pruebas de backend superadas.
- Cobertura global medida del backend: **59%**.
- Seis pruebas nuevas para datasets, seguridad, persistencia e informe.
- La compilacion Python se valida con `compileall`.
- El frontend se construye en Docker y en GitHub Actions mediante `npm ci` y
  `npm run build`.

## Actualizacion

```powershell
Set-ExecutionPolicy -Scope Process Bypass
.\scripts\upgrade-v0.11.ps1
```

No utilizar `docker compose down -v`.

## Reversion

El codigo puede volver a v0.10.x manteniendo la tabla nueva. Si fuera necesario
revertir tambien el esquema:

```powershell
docker compose -f docker-compose.yml -f docker-compose.gpu.yml exec api alembic downgrade 0008_security_audit
```

La reversion de esquema elimina exclusivamente los informes de evaluacion
formal almacenados en `system_evaluation_runs`.

## Limitaciones conocidas

- La puntuacion automatica no sustituye la revision de un veterinario.
- Ejecutar pytest desde la aplicacion bloquea temporalmente esa peticion; se ha
  aceptado para el MVP local y no se recomienda como patron de produccion.
- El tiempo de evaluacion de VetIA depende de Ollama, Qdrant y la GPU disponible.
