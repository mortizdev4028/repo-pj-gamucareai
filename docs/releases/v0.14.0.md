# GamuCare AI 0.14.0

## Objetivo
Cerrar el bloque de calidad del RAG, eliminar falsos positivos fuera del dominio
veterinario y preparar un corpus oficial mas amplio y trazable sin redistribuir
documentos completos de terceros dentro del proyecto.

## Cambios funcionales
- VetIA rechaza preguntas fuera del ambito veterinario antes de buscar documentos.
- Los rechazos distinguen `out_of_scope`, `low_score` y `no_evidence`.
- G12, capital de Australia, y C07, placas base de ordenadores, dejan de generar resultados.
- Calidad de VetIA muestra las fuentes externas disponibles, los errores de la
  ultima descarga y el motivo de cada caso que no supera la evaluacion.
- El dataset activo pasa a `rag_cases_v2.json` con 23 casos.

## Cambios tecnicos
- Analisis de dominio determinista en `app.services.rag_intelligence`.
- Comparacion por palabras completas y variantes plurales simples.
- No se solicita embedding ni se invoca Qdrant cuando la decision es
  `out_of_scope`.
- `RagDiagnostics` incorpora decision, razon de dominio y puntuacion del mejor
  candidato antes del umbral.
- Ingesta de Markdown, TXT, PDF y HTML.
- Extraccion de PDF con `pypdf` y limpieza de HTML con `BeautifulSoup`.
- Catalogo de siete fuentes externas oficiales.
- Descarga por HTTPS con limite de 25 MiB, hash SHA-256 y sidecar JSON.

## Frontend
- Estado de corpus externo en Calidad de VetIA.
- Motivo y decision de recuperacion en la tabla de revision.
- Resumen de los tipos de rechazo registrados durante la evaluacion.
- Version visible actualizada a 0.14.0.

## Backend y API
- Sin endpoints nuevos.
- `/api/v1/rag/status` anade:
  - `source_manifest_total`.
  - `external_files_available`.
  - `external_download_failures`.
- La respuesta de diagnostico RAG anade campos de decision de recuperacion y dominio.

## Base de datos
No hay cambios de esquema ni nueva migracion Alembic. La ultima migracion sigue
siendo `0009_system_evaluation`.

La reindexacion modifica la coleccion de Qdrant, pero no elimina datos de
PostgreSQL.

## Variables de entorno nuevas
```text
RAG_EXTERNAL_DOCUMENTS_PATH=/app/data/rag_external
RAG_SOURCE_MANIFEST=/app/data/rag_sources/sources.json
RAG_SOURCE_MAX_BYTES=26214400
```

Los valores por defecto ya estan incluidos en Docker Compose y `.env.example`.
Un `.env` existente puede conservarse sin cambios.

## Datos y fuentes
- Se conserva `rag_cases_v1.json` para trazabilidad historica.
- Se incorpora `rag_cases_v2.json` como dataset activo.
- Los documentos externos no estan dentro del ZIP.
- `scripts/download-rag-sources.ps1` los descarga desde el organismo de origen.
- Los ficheros se almacenan en `data/rag_external` con metadatos y hash.

## Actualizacion desde 0.13.x
```powershell
Set-ExecutionPolicy -Scope Process Bypass
.\scripts\upgrade-v0.14.ps1
```

El script reconstruye API y frontend, conserva los volumenes, aplica las
migraciones existentes, reindexa Qdrant, ejecuta las pruebas y arranca la
monitorizacion.

Para descargar las siete fuentes externas durante la actualizacion:

```powershell
.\scripts\upgrade-v0.14.ps1 -DownloadSources
```

Tambien pueden descargarse despues:

```powershell
.\scripts\download-rag-sources.ps1 -Reindex
```

## Rollback
1. Detener API y web.
2. Restaurar los ficheros de la version 0.13.0.
3. Conservar los volumenes de PostgreSQL, Qdrant y Ollama.
4. Reconstruir API y frontend con los ficheros 0.13.0.
5. Ejecutar `python -m app.rag.ingest` desde la API 0.13.0 para reconstruir la
   coleccion con el comportamiento anterior.

No es necesario revertir una migracion de PostgreSQL.

## Problemas conocidos
- La descarga de fuentes necesita conectividad a Internet y depende de que la URL
  oficial siga disponible.
- Los PDF escaneados sin texto no utilizan OCR y se omiten si no se puede extraer
  contenido suficiente.
- Parte del corpus externo esta en ingles; la recuperacion multilingue debe
  revisarse durante la evaluacion.
- El filtro de dominio es determinista y puede necesitar nuevos terminos si se
  incorporan especialidades veterinarias adicionales.
- El bundle principal de Vite supera 500 kB y genera una advertencia de tamano;
  no impide la compilacion ni el funcionamiento del MVP.
