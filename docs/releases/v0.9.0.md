# GamuCare AI 0.9.0 - Integracion simulada con Wakyma

## Objetivo

La version 0.9.0 convierte el importador inicial de JSON en un conector simulado completo, auditable y utilizable desde la interfaz web. El objetivo no es afirmar que existe una API publica de Wakyma, sino demostrar como GamuCare recibiria y normalizaria informacion de un sistema de gestion veterinaria.

## Cambios funcionales

### Nueva pantalla de integracion

Los perfiles `clinic` y `staff` disponen de la pantalla **Integracion Wakyma**.

- Descarga de plantillas JSON y CSV.
- Seleccion de ficheros desde el navegador.
- Validacion previa sin modificar datos.
- Importacion real reservada a `clinic`.
- Resumen de registros creados, actualizados y rechazados.
- Historial de ejecuciones.
- Detalle por fila con entidad, identificador externo, accion, resultado y mensaje.
- Descarga puntual de credenciales temporales cuando se crean propietarios.

### Entidades soportadas

- Propietarios.
- Mascotas.
- Eventos clinicos.

Las relaciones se resuelven mediante identificadores externos. Una mascota referencia a su propietario y un evento clinico referencia a su mascota.

### Actualizacion idempotente

El campo `external_id` decide si un registro se crea o se actualiza. Importar dos veces el mismo fichero no duplica clientes, mascotas ni eventos; la segunda ejecucion actualiza los registros existentes.

### Validacion parcial

Un registro erroneo no invalida necesariamente el fichero completo. Los registros independientes correctos se procesan y los fallos quedan documentados por fila.

Se validan, entre otros:

- Formato y version del fichero.
- Campos obligatorios.
- Correo electronico.
- Especie y sexo.
- Fechas ISO.
- Peso positivo.
- Valores booleanos.
- Duplicados dentro del fichero.
- Referencias entre propietario, mascota y evento.
- Conflictos de correo con usuarios existentes.

### Sincronizacion posterior

Tras una importacion real, cada mascota afectada se envia a un proceso en segundo plano que:

1. Actualiza su ficha y sus eventos en Qdrant.
2. Recalcula las reglas preventivas.
3. Enriquece con RAG los avisos activos que lo requieran.

La importacion de PostgreSQL no se revierte si Ollama o Qdrant no estan disponibles; el fallo de sincronizacion queda en los logs y puede repetirse posteriormente.

## Seguridad y permisos

| Operacion | Clinica | Personal |
|---|---:|---:|
| Consultar estado e historial | Si | Si |
| Descargar plantillas | Si | Si |
| Validar un fichero | Si | Si |
| Ejecutar una importacion | Si | No |

Los propietarios no tienen acceso al modulo.

Las contrasenas temporales de cuentas nuevas se generan aleatoriamente, se almacenan mediante Argon2 y solo se devuelven en la respuesta de la importacion. El texto plano no se guarda en la auditoria.

## Cambios de base de datos

Migracion Alembic `0007_wakyma_integration`.

### Ampliacion de `import_batches`

- Usuario que solicita la operacion.
- Formato y version del esquema.
- Checksum SHA-256.
- Indicador de validacion previa.
- Totales de creados, actualizados, omitidos y fallidos.
- Resumen estructurado.

### Nueva tabla `import_batch_items`

Registra el resultado de cada fila con su entidad, identificador externo, accion prevista o ejecutada, estado, mensaje y carga normalizada.

## Cambios tecnicos

- Nuevo servicio `app.integrations.wakyma.service`.
- Parser comun para JSON y CSV.
- Separacion entre parseo, validacion y persistencia.
- El adaptador de linea de comandos reutiliza el mismo servicio.
- Dependencia `python-multipart` para cargas de fichero en FastAPI.
- Metricas Prometheus:
  - `gamucare_wakyma_imports_total`
  - `gamucare_wakyma_import_duration_seconds`
  - `gamucare_wakyma_records_total`


No utilizar `docker compose down -v`.

## Impacto sobre datos existentes

La migracion conserva todos los datos actuales. Las importaciones antiguas permanecen en `import_batches` y reciben valores por defecto en los nuevos campos.

## Limitaciones conocidas

- No existe conexion real a Wakyma en esta version.
- El conector trabaja con ficheros y no realiza sincronizacion incremental programada.
- La resolucion de conflictos es automatica por `external_id`; no existe todavia una pantalla para fusion manual.
- Las importaciones se ejecutan dentro de la peticion HTTP, por lo que el limite se mantiene en 5 MB para el MVP.

## Reversion

Antes de revertir codigo, aplicar:

```powershell
docker compose -f docker-compose.yml -f docker-compose.gpu.yml exec api alembic downgrade 0006_rag_quality
```

La reversion elimina el detalle de auditoria de la v0.9.0 y las columnas nuevas de `import_batches`, pero no elimina clientes, mascotas o eventos que ya hayan sido importados.
