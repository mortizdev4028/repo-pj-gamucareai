# GamuCare AI 0.15.0

## Objetivo
Cerrar la revision funcional y endurecer el MVP local antes del candidato de
entrega, sin anadir nuevas funciones de negocio ni recuperar el alcance AWS.

## Cambios funcionales
- Los accesos no autorizados muestran una pantalla explicita con el perfil y la ruta solicitada.
- Un fallo inesperado de renderizado muestra una recuperacion controlada en lugar de una pantalla en blanco.
- Los mensajes API pueden incluir el identificador de solicitud para facilitar soporte.

## Backend y API
- Respuesta compatible de error con `detail`, `error_code` y `request_id`.
- Cabecera `X-Request-ID` en respuestas correctas y fallidas.
- Respuesta 500 generica sin detalles internos.
- Validaciones 422 sin reproducir los valores enviados.
- `Retry-After: 15` en respuestas 503 generadas mediante `HTTPException`.

Ejemplo:

```json
{
  "detail": "No tienes permisos para esta operacion",
  "error_code": "forbidden",
  "request_id": "5f4d..."
}
```

## Frontend
- Nueva ruta `/forbidden`.
- Nuevo `AppErrorBoundary` global.
- Utilidad comun para mostrar errores API y su identificador de solicitud.
- Separacion del bundle en `react-vendor`, `mui-vendor`, `http-vendor` y codigo de aplicacion.

## Automatizacion y operacion
- `test-permissions.ps1`: 22 comprobaciones de acceso.
- `resilience-test.ps1`: caida y recuperacion de Qdrant y Ollama.
- `release-validation.ps1`: orquesta smoke test, permisos, pytest, evaluacion, resiliencia y backup opcional.
- `upgrade-v0.15.ps1`: actualizacion conservando volumenes.
- CI Windows para validar la sintaxis de los scripts PowerShell.

## Base de datos
No hay cambios de esquema. La migracion de cabecera sigue siendo
`0009_system_evaluation`.

## Variables de entorno
No hay variables nuevas obligatorias.

## Actualizacion desde 0.14.x

```powershell
Set-ExecutionPolicy -Scope Process Bypass
.\scripts\upgrade-v0.15.ps1
```

Opciones de cierre durante la actualizacion:

```powershell
.\scripts\upgrade-v0.15.ps1 -RunPermissionMatrix
.\scripts\upgrade-v0.15.ps1 -IncludeResilience
```

La matriz no se ejecuta por defecto porque puede fallar si se han cambiado las
contrasenas de demostracion. La resiliencia tampoco se ejecuta por defecto
porque detiene Qdrant y Ollama temporalmente.

## Validacion recomendada

```powershell
.\scripts\release-validation.ps1 -IncludeMonitoring
```

Validacion ampliada:

```powershell
.\scripts\release-validation.ps1 -IncludeMonitoring -IncludeResilience -IncludeBackup
```

## Rollback
1. Detener `api` y `web`.
2. Restaurar los ficheros de la version 0.14.0.
3. Conservar los volumenes PostgreSQL, Qdrant y Ollama.
4. Reconstruir `api` y `web`.

No es necesario ejecutar `alembic downgrade`.

## Problemas conocidos
- La matriz de permisos utiliza una contrasena comun por parametro; si cada usuario
  tiene una clave distinta, debe ejecutarse manualmente o adaptar el script.
- La prueba de resiliencia produce una caida real y temporal de Qdrant y Ollama.
- No se automatiza la caida de PostgreSQL para evitar riesgos sobre la base transaccional.
- La validacion completa con documentos externos y generacion depende de Ollama,
  Qdrant y del corpus descargado en la instalacion del usuario.
