# GamuCare AI 0.16.0-rc1

## Objetivo

Crear el candidato de entrega del TFM. El alcance queda congelado, AWS se
excluye y no se incorporan nuevas funciones de negocio. La prioridad pasa a ser
reproducibilidad, evidencias, documentacion y demostracion.

## Cambios funcionales

- La interfaz muestra `DEMO` y `Candidato de entrega` para evitar confundir el
  entorno academico con una aplicacion de produccion.
- Se mantiene la separacion entre clinica, personal, propietario y tecnico.
- No se modifican clientes, mascotas, planes, pagos, avisos ni comportamiento de VetIA.

## Dataset de demostracion

Nuevo comando destructivo y confirmado:

```powershell
.\scripts\reset-demo.ps1
```

Reconstruye:

1. Esquema PostgreSQL y migraciones.
2. Usuarios, propietarios, mascotas, planes, cuotas e historiales ficticios.
3. Casos clinicos recurrentes de demostracion.
4. Coleccion Qdrant.
5. Avisos preventivos y explicaciones.
6. Evaluaciones y resumen agregado.

Conserva los modelos de Ollama y los volumenes de monitorizacion. No equivale a
`reset.ps1`, que elimina todos los volumenes.

## Evidencias

```powershell
.\scripts\generate-evidence.ps1 -IncludeMonitoring -IncludePermissions
```

Genera una carpeta fechada con:

- Estado de Docker Compose.
- Health checks y dependencias.
- Pytest, cobertura y JUnit.
- Calidad de VetIA.
- Validacion del MVP.
- Resumen agregado de release.
- Targets y alertas de Prometheus, cuando se solicita.
- Matriz de permisos, resiliencia y backup como opciones.
- `files.sha256`.

## Resumen agregado

Nuevo modulo:

```powershell
docker compose -f docker-compose.yml -f docker-compose.gpu.yml exec -T api `
  python -m app.scripts.release_snapshot
```

Incluye conteos, estado de evaluaciones y dependencias. No exporta direcciones,
telefonos, correos de propietarios ni textos clinicos.

## Actualizacion desde 0.15.x

```powershell
Set-ExecutionPolicy -Scope Process Bypass
.\scripts\upgrade-v0.16-rc1.ps1
```

Opciones:

```powershell
.\scripts\upgrade-v0.16-rc1.ps1 -DownloadSources
.\scripts\upgrade-v0.16-rc1.ps1 -ResetDemo
.\scripts\upgrade-v0.16-rc1.ps1 -GenerateEvidence
```

`-ResetDemo` destruye los datos actuales de PostgreSQL; debe utilizarse solo en
un entorno de demostracion.

## Rollback

1. Detener API y frontend.
2. Restaurar los ficheros de v0.15.0.
3. Mantener los volumenes si no se ejecuto `reset-demo.ps1`.
4. Reconstruir API y frontend.

No se requiere `alembic downgrade`.

## Problemas conocidos

- El paquete de evidencias debe ejecutarse en Docker Desktop; no contiene
  resultados precalculados.
- La evaluacion con generacion depende de la latencia y disponibilidad de Ollama.
- La restauracion real debe probarse en una copia antes de la version 1.0.0.
- El candidato no esta certificado para produccion ni para uso clinico real.
