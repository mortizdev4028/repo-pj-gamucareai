# Historial consolidado de versiones

| Version | Cambio principal |
|---|---|
| 0.1.0 | Primera arquitectura con React, FastAPI, PostgreSQL, Qdrant, Ollama, planes LifeCare y datos ficticios. |
| 0.1.1 | Correccion de la compilacion TypeScript. |
| 0.2.0 | Tres perfiles: clinica, personal de solo lectura y propietario; gestion de clientes y mascotas. |
| 0.2.1 | Eliminacion de URLs internas del `package-lock.json`. |
| 0.3.0 | RAG con fichas e historiales clinicos y explicaciones de avisos. |
| 0.4.0 | Consultas privadas por mascota y seguimiento economico de los planes. |
| 0.5.0 | Asignacion, cambio, cancelacion, renovacion, solicitudes y calendario de cuotas. |
| 0.6.0 | Catalogo versionado, recurrencias, ciclo de vida, auditoria y panel completo de avisos preventivos. |
| 0.7.0 | Base documental ampliada, reranking explicable, citas, contexto conversacional y evaluacion versionada del RAG. |
| 0.8.0 | Paneles por perfil, indicadores operativos y economicos, tendencias, agenda y exportacion CSV. |
| 0.9.0 | Integracion simulada Wakyma con JSON/CSV, validacion previa, upsert, auditoria y sincronizacion con RAG y avisos. |
| 0.10.0 | Politica de contrasenas, sesiones renovables y revocables, bloqueo de cuenta, auditoria transversal y cabeceras de seguridad. |
| 0.11.0 | Evaluacion formal del MVP, cobertura, datasets controlados, informes descargables, version visible y nombre VetIA en la interfaz. |
| 0.12.0 | Observabilidad local, alertas tecnicas, logs rotados, backups, restauracion, smoke tests y pipeline de imagenes. |
| 0.13.0 | Perfil tecnico aislado, menu y endpoints tecnicos exclusivos, y bloqueo cruzado con la operacion clinica. |
| 0.14.0 | Filtro de dominio, rechazo explicable, corpus externo trazable e ingesta de PDF y HTML. |
| 0.15.0 | Cierre funcional, errores trazables, matriz de permisos, resiliencia controlada y validacion de release. |
| 0.16.0-rc1 | Candidato de entrega, entorno demo reproducible, evidencias automaticas y documentacion final consolidada. |

El detalle de cada entrega se encuentra en `CHANGELOG.md` y en `docs/releases/`.
